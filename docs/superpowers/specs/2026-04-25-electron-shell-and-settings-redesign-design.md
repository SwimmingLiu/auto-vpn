# Electron 外壳精简与设置抽屉重构设计

## 背景

当前 Electron 客户端已经具备六页工作区和基础交互，但存在几类明显的 UI/UX 问题：

- 订阅页的格式切换按钮仅是静态文案，`Clash`、`Clash Meta`、`Sing-box`、`Surge` 不会驱动 URL、二维码与操作按钮。
- 左侧菜单栏底部长期显示滚动条，图标与字体偏小；底部“系统状态”与首页内容重复。
- 顶部快捷导航与左侧主导航重复，占据了最显眼的空间。
- 首页的“高频操作”卡片价值低，且与顶部/运行页的控制能力重叠。
- 日志页的筛选按钮没有真实绑定，操作按钮也没有进入同一工具栏。
- 设置页仍是平铺大表单，“编辑”按钮没有承担清晰职责，缺少桌面应用常见的分区、抽屉、取消/保存节奏。

## 目标

本轮改造只做前端壳层与交互重构，不改变后端流水线协议。

1. 精简整体导航层级：左侧只保留主导航，顶部只保留当前页相关操作。
2. 让订阅页的格式切换真正生效，并改成更紧凑的布局。
3. 去掉左侧系统状态卡，把必要状态并入首页顶部状态区。
4. 让日志页具备真实筛选能力，并把日志操作纳入同一行工具栏。
5. 把设置页改为“分组卡片 + 右侧抽屉编辑 + 全局保存”的桌面工具模式。
6. 保持现有浅色、蓝紫主色、圆角玻璃化的视觉主题不变。

## 非目标

- 不重写 Python backend。
- 不调整流水线阶段定义与 IPC 事件结构。
- 不引入新的前端框架。
- 不做独立安装包形态改动。

## 方案比较

### 方案 A：最小修补

- 继续保留现有页面骨架；
- 只补按钮绑定、隐藏滚动条、删掉重复文案；
- 设置页仍保留平铺表单。

优点：改动小。  
缺点：界面层级问题基本不变，设置页体验仍然重。

### 方案 B：壳层精简 + 局部重构（推荐）

- 删除顶部快捷导航条；
- 左侧保留主导航并放大图标/字重；
- 首页合并系统状态，移除“高频操作”；
- 订阅页、日志页、设置页分别做局部重构；
- 设置页改为卡片概览 + 右侧抽屉编辑；
- 顶部操作条按页面动态切换。

优点：能直接覆盖这次反馈中的六类问题，且不需要改后端协议。  
缺点：前端状态与测试会有一轮集中调整。

### 方案 C：整站页面重画

- 重新定义六页布局、卡片体系和所有组件；
- 同时调整结果页和运行页的大部分视觉结构。

优点：视觉统一度最高。  
缺点：范围过大，和当前用户反馈不成比例，回归成本高。

## 选型

采用 **方案 B**。

这是当前投入产出比最高的路线：结构明显变干净，交互变真实，同时仍能复用现有 `views.js` / `app.js` / `styles.css` 组织方式和现有测试框架。

## 交互决策

本次设计采用以下已确定决策：

- 设置页使用 **右侧抽屉**。
- 订阅页格式按钮采用 **切换格式后同步更新 URL / 二维码 / 复制 / 打开目标**。
- 顶部栏采用 **按页面动态变化的操作按钮**。
- 日志筛选采用 **纯前端筛选**：全部 / 运行日志 / 错误 / 按阶段。
- 首页将左侧“系统状态”并入 **顶部主状态区**。

## 信息架构调整

### 左侧导航

- 保留六个主页面入口：概览 / 运行 / 结果 / 订阅 / 日志 / 设置。
- 去掉底部系统状态卡。
- 调整为更宽的导航按钮、更大的图标容器与字重。
- 不再出现可见滚动条；即便后续内容溢出，也使用隐藏滚动条样式。

### 顶部栏

- 保留页面标题、运行态 badge。
- 删除原 `shortcutStrip` 整块快捷导航。
- 右侧操作区改为 `page actions`，不同页面展示不同按钮：
  - 概览：开始运行 / 停止运行 / 打开设置
  - 运行：开始运行 / 停止运行 / 重试当前阶段
  - 结果：打开 artifact / 复制节点
  - 订阅：复制订阅 / 打开订阅
  - 日志：不在顶部放日志操作，日志操作全部下沉到页内工具栏
  - 设置：保存配置

## 页面级设计

### 1. 首页（概览）

- 删除“高频操作”卡片。
- 保留“当前运行状态”“指标卡”“最近运行结果”。
- 在状态区新增紧凑的“系统状态摘要”，承接原左侧状态卡中的：
  - 状态
  - 模式
  - 最后结果
  - 最后更新
  - 当前任务

首页的角色变成：**看状态 + 看结果**，不再承担重复控制台职责。

### 2. 订阅页

- 顶部仍展示主订阅地址。
- 原格式按钮改成紧凑分段控件，按钮尺寸更小、更收拢。
- 当前选中的格式驱动：
  - 输入框里的订阅 URL
  - 二维码内容
  - “复制链接”按钮
  - “打开订阅”按钮
- `Clash Meta`、`Sing-box`、`Surge` 的格式参数通过已有 `buildSubscriptionCards` 规则生成。

### 3. 日志页

- 顶部变成一行工具栏：
  - 左侧：全部 / 运行日志 / 错误 / 按阶段
  - 右侧：复制日志 / 清空显示 / 打开日志文件
- “运行日志”显示普通过程日志，过滤掉错误行。
- “错误”只显示错误/失败/异常相关行。
- “按阶段”不是单纯列表，而是按照阶段标题分组显示；无法识别阶段的日志进入“其他”组。

### 4. 设置页

- 主页面不再直接摊开所有输入框。
- 改成若干概览卡片：
  - 数据源配置
  - 测速配置
  - 部署配置
  - 路径配置
  - 关于
- 点击卡片后，从右侧拉出抽屉。
- 抽屉内编辑当前分组字段，底部固定 `取消` / `保存`。
- 抽屉的 `保存` 只把当前分组写回前端 profile 草稿；真正落盘由顶部全局 `保存配置` 完成。
- 抽屉的 `取消` 丢弃当前分组未保存编辑。

## 状态模型调整

前端状态从“页面 + 通用 subtab”扩展为更明确的页面状态：

- `activePage`
- `subscriptionFormat`
- `logFilter`
- `settingsDrawer`
  - `section`
  - `draft`
  - `dirty`

其中：

- `subscriptionFormat` 默认 `Clash`
- `logFilter` 默认 `全部`
- `settingsDrawer` 在关闭时为 `null`

这能避免继续把“订阅格式切换”“日志筛选”“设置抽屉编辑”都塞进一个模糊的 `subtabs` 对象。

## 日志筛选与分组策略

日志数据会扩展成结构化条目：

- `line`
- `level`
- `stage`
- `kind`

来源规则：

1. `handlePipelineEvent({ type: 'log' })` 继续写入普通日志。
2. `handlePipelineEvent({ type: 'stage' })` 同时写入一条结构化阶段日志，确保“按阶段”一定有可分组内容。
3. 通过关键字推断 `level`：
   - error / failed / exception / traceback / 错误 / 失败 → `error`
   - warn / warning / 警告 → `warning`
   - 其他 → `info`
4. 通过阶段名或已有 `stage` 事件推断 `stage`，无法识别则归到 `other`。

## 代码组织

### `electron/renderer/index.html`

- 删除侧栏状态卡 DOM。
- 删除 `shortcutStrip` DOM。
- 保留一个更简化的顶部 `pageActions` 容器。
- 增加设置抽屉容器和遮罩容器。

### `electron/renderer/app.js`

- 增加：
  - `subscriptionFormat`
  - `logFilter`
  - `settingsDrawer`
- 新增点击处理：
  - 订阅格式切换
  - 日志筛选切换
  - 清空日志
  - 打开日志文件
  - 打开/关闭设置抽屉
  - 抽屉保存/取消
- `renderChrome()` 改为渲染动态顶部操作区，不再渲染 `shortcutStrip` 与左侧状态卡。
- `appendLog()` 改为写入结构化日志条目。

### `electron/renderer/views.js`

- 删除 `buildShortcutStrip()`、`buildSidebarStatus()` 的实际使用。
- 增加：
  - `buildTopbarActions()`
  - `buildSettingsCards()`
  - `buildSettingsDrawer()`
  - `filterLogRows()`
  - `groupLogRowsByStage()`
- 首页、订阅页、日志页、设置页分别重写局部 markup。

### `electron/renderer/styles.css`

- 调整：
  - 侧栏宽度、图标尺寸、字重、间距
  - 侧栏滚动条隐藏
  - 顶部栏双栏布局
  - 订阅页紧凑分段控件

  - 设置卡片 + 抽屉 + 遮罩

## 验证策略

1. Node test 覆盖新的页面交互与状态切换。
2. Playwright e2e 验证：
   - 顶部快捷导航已移除
   - 订阅格式切换真的改变 URL
   - 日志筛选可切换
   - 设置抽屉可打开、取消、保存
3. Playwright 视觉快照更新六页 hash。
4. 打包后实际打开 Electron 应用，并用 Computer Use 检查：
   - 左侧无可见滚动条
   - 顶部不再与左侧导航语义重叠
   - 设置抽屉风格和主题一致

## 风险与处理

### 风险 1：日志来源不够结构化

处理：在前端为 `stage` 事件补充合成日志，确保“按阶段”最少可用。

### 风险 2：设置抽屉保存语义混乱

处理：明确区分“抽屉保存到前端草稿”和“顶部保存到磁盘”。

### 风险 3：视觉回归测试大面积失效

处理：把 e2e 行为测试先修稳，再更新 visual hash，最后做实际窗口检查。
