# Electron Compact Dashboard Design

## Outcome

把 Electron 首页调整为“半屏优先”的紧凑控制台：默认以约 `960x720` 的 MacBook 非全屏窗口为目标，在不全屏时也能完整看完首页核心功能，不再依赖全屏展示。

## Goals

- 首页在 `960x720` 档位下无横向遮挡、无纵向裁切，核心信息一屏可见
- 默认视觉更紧凑，窗口变大时再逐步增加留白，而不是始终铺满
- 只展示最终用户需要的功能说明，不再暴露开发阶段沟通文案
- 保留现有抽屉式详细参数编辑能力，但抽屉不能造成主界面明显遮挡或越界感

## Layout Direction

### 1. 外层容器

- 主容器从“占满整个窗口”改为“居中展示”
- 大窗口下限制最大宽度，让页面从屏幕中部向外扩张，维持约半屏到六成屏宽的观感
- 边距、圆角、阴影、模糊效果同步收敛，整体更接近工具台而不是宣传页

### 2. 顶部工具栏

- 保留品牌、语言切换、保存、运行四个核心操作
- 压缩高度、按钮尺寸和控件间距，避免顶部区域过高
- 标题与副标题保留，但字号降低一级，强调“可用性”而不是“展示感”

### 3. 概览区

- 现有高占比 hero 改成短摘要概览区
- 只保留一句产品级描述，说明该工具用于节点抓取、测速和发布
- 不再展示任何类似“无需全屏”“展开抽屉查看参数”这类开发过程说明

### 4. 主工作区

- 左侧改成紧凑摘要网格，承载：
  - 抓包源
  - 测速配置
  - 部署配置
  - 运行指标
- 右侧保留两块纵向信息：
  - 阶段状态
  - 日志摘要
- 这两个区域在半屏尺寸下仍应完整显示，不出现底部卡片被压到只剩标题的情况

### 5. 抽屉

- 抽屉保留右侧滑出方式
- 宽度略收窄，并确保在非全屏窗口下仍有明确边界和安全留白
- 抽屉内部允许滚动，但页面主体不应因为抽屉出现而产生新的遮挡或布局抖动

## Responsive Rules

- 目标基线：`960x720`
  - 首页默认展示完整，不出现滚动条依赖
  - 维持双列，但采用更紧凑的卡片尺寸和更低的模块高度
- 更大窗口：
  - 逐步放大容器宽度、模块间距和内部留白
  - 视觉层次增强，但不回到“全宽铺满”的展示型布局
- 更窄窗口：
  - 在接近单栏阈值时允许上下堆叠
  - 进入单栏后仍需保持顶部、摘要区、状态区、日志区的顺序清晰

## Copy Rules

- 首页文案只能描述产品能力、配置对象和运行状态
- 移除所有面向开发过程的措辞，包括但不限于：
  - 关于是否全屏的说明
  - 关于首页信息架构的自我解释
  - 关于“详细参数通过展开式抽屉查看”之类的开发说明
- 中英文文案都同步收敛为用户可读的产品表达

## Verification

- Electron renderer e2e:
  - 以 `960x720` 视口验证无横向/纵向溢出
  - 校验关键模块都在首屏可见
- Electron visual regression:
  - 更新紧凑布局视觉快照
- 文案验证:
  - 断言被用户指出的开发说明文案不再出现在首页

## Out of Scope

- 不改变 Python backend 流水线能力
- 不新增新的业务功能或导航结构
- 不把首页改成多页签工作流
