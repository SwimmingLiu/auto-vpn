# Electron 运行时对齐与默认空态设计

## 目标

修复当前 Electron 桌面端的三个结构性问题：

1. 渲染层过度贴合设计稿，出现大量没有接真实行为的按钮与页面。
2. 没有真实数据时仍然展示伪造的统计、历史、节点、监控和订阅样本，导致界面误导。
3. 打包后的 `.app` 只能启动 Electron 外壳，无法稳定加载 Python backend 与既有配置。

本轮目标不是继续“补齐设计稿上的所有假功能”，而是把界面收束到真实能力，并让打包产物可实际运行。

## 用户新增约束

- 设计稿仅作为视觉参考，不是最终功能分布的强约束。
- 页面初始应显示默认空态，而不是内置样例业务数据。
- 之前已经保存过的配置，应在 Electron 开发态与打包态里自动加载。
- 不支持的动作不能再伪装成可点击可操作按钮。

## 当前真实能力

现有项目真正具备的能力是：

- 加载 / 保存 `AppProfile`
- 运行流水线
- 停止流水线
- 接收实时日志
- 接收阶段状态与 summary
- 编辑抓包源、测速参数、部署参数
- 输出 artifacts 到本地目录

当前**没有**独立实现的能力包括：

- 历史任务持久化查询
- 节点独立管理
- 订阅地址多客户端管理中心
- 系统监控采集
- 暂停 / 继续运行
- 独立部署任务入口

因此这些内容不应该继续以“已有完整功能”的形态出现。

## 设计决策

### 1. 页面收束为真实功能页

导航收束为 6 个主页面：

1. `dashboard`：总览、最近运行摘要、空态提示、快捷入口
2. `config`：抓包源 / 测速 / 部署配置编辑
3. `run`：运行控制、阶段状态、实时日志
4. `artifacts`：当前订阅地址、产物文件、输出目录操作
5. `logs`：当前会话日志浏览、过滤、导出
6. `about`：版本、工作目录、运行时说明

这样可以把动作放回真实位置：

- 保存配置只在配置上下文中突出显示
- 运行 / 停止只在运行页和全局顶栏出现
- 打开输出目录只在 artifacts 页出现
- 日志导出只在 logs 页出现

### 2. 默认空态

如果没有运行 summary、没有 artifacts、没有 session logs，则显示：

- `暂无运行数据`
- `尚未生成订阅地址`
- `尚未生成产物文件`
- `尚未采集日志`

统计卡片统一回退到 `0 / — / 未运行`，不再注入样例数字。

如果没有保存过 profile，则默认 profile 为真正空白值：

- source url / key 为空
- deploy.project_name / subscription_url 为空
- speed_test 使用最小结构，但不展示业务样例 URL

### 3. 不支持的动作处理

对未实现能力采用两类处理：

- **直接移除**：暂停、继续、终止、系统监控、历史分析等
- **保留但禁用且明确说明**：如果界面结构需要占位，则显示禁用态和说明文本

默认优先移除，而不是展示“未来可能有”的假按钮。

### 4. 打包态运行时

打包态必须满足两件事：

1. Python backend 源码可被系统 Python 导入
2. 运行时 profile 从可写目录读取，并在首次打包运行时迁移旧配置

方案：

- Electron 打包配置纳入 `src/**/*`、`pyproject.toml`、以及 profile seed 文件
- 关闭 `asar`，避免 Python 无法导入 `app.asar/src`
- Electron main 在 packaged 模式下把 profile 读写位置切到用户可写目录
- backend 通过环境变量接收 profile 路径
- 首次 packaged 启动时，如果用户目录里没有 profile，则：
  - 先尝试复制 bundled seed profile
  - 再回退到空白 profile

### 5. 既有配置自动加载

优先级：

1. Electron 传入的显式 runtime profile path
2. 用户目录中的已保存 profile
3. bundled seed profile（打包时带入）
4. 空白默认 profile

这能同时覆盖：

- 开发态：继续读取仓库 state
- 打包态首次运行：迁移现有配置
- 打包态后续运行：读取用户目录配置

## 验收标准

- 主导航只保留真实功能页面。
- 页面初始化不再出现伪造业务数据。
- 未实现能力不再展示为可用按钮。
- 已存在 profile 能在开发态自动加载。
- 打包 `.app` 启动后可成功执行 `profile:load`。
- 打包 `.app` 在首次运行时能从 bundled seed 或旧配置迁移得到 profile。
- Playwright e2e、视觉回归、Python tests、Electron tests、打包后真机启动验证全部通过。
