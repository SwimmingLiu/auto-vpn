# AutoVPN 移动端 UX/UI 重构设计

## 背景与目标

AutoVPN 的 H5 界面目前在 720px 以下切换为底部导航，但主体仍大量沿用桌面端布局与交互。实际审查确认设置表格、运行控制、全屏抽屉、焦点管理、横屏和安全区存在结构性问题；当前移动测试也只覆盖 390×844 下的概览、运行和日志页面。

本次重构要求移动端与桌面端保持完整功能一致，同时让手机和平板上的导航、运行、结果、订阅、日志和全部设置都可读、可触控、可完成任务。桌面端的信息密度和工作流不得退化。

## 使用场景

用户在手机上通过浏览器远程查看自动化状态，可能在通勤、弱网或单手操作场景中快速启动/停止任务、复制订阅、检查失败原因，偶尔需要完成完整配置。界面应优先呈现当前状态与下一步操作，复杂设置通过渐进展开完成，而不是要求用户切换到桌面端。

## 方案选择

### 采用：共享组件与状态，按上下文自适应重排

- 复用现有状态、事件与数据结构。
- 桌面端继续使用侧栏、数据表和宽面板。
- 手机端使用底部导航、单列内容、紧凑任务操作条、字段列表和全屏 sheet。
- 平板端使用带文字的顶部导航和一至两列布局。
- 用语义化组件状态解决焦点、读屏、软键盘和触摸问题，而非仅靠媒体查询缩小尺寸。

### 未采用：只补响应式 CSS

改动较小，但无法可靠解决隐藏可聚焦元素、模态语义、设置表格字段关系、日志增量更新和二维码错误状态等结构问题。

### 未采用：独立移动端页面

可以获得最大布局自由，但会复制六个页面及行为逻辑，长期容易与桌面功能漂移，测试与维护成本过高。

## 信息架构与导航

### 手机（320–720px）

- 保留六个一级入口，固定底部导航；每项至少 48×48px，图标与短标签同时显示。
- 320–380px 不缩小字体；导航允许水平滚动并自动保持当前项可见，避免截断和误触。
- 页面头部只保留标题、运行状态与当前页面最重要的一个操作；次级操作进入页面内操作区。
- 页面内容增加底部安全间距，任何最后一个控件都能滚动到导航上方。

### 平板（721–960px）

- 使用顶部横向导航，保留图标与文字标签。
- 依据内容在一列与两列间切换，不沿用仅图标的 90px 桌面窄侧栏。

### 桌面（961px 以上）

- 保持现有侧栏与高密度布局，不因移动端重构降低信息量。

## 页面设计

### 概览

- 首屏顺序为运行状态、主要操作、四项关键计数、最近运行和系统状态。
- 计数使用一个分段指标面板，移除“外层卡片套四张卡片”的重复边框与阴影。
- 数据源明细以可换行的紧凑标签呈现。

### 运行

- 只将“开始/停止”双操作条做成 `position: fixed` 的底部 action surface，始终位于底部导航上方；运行页其他控件不得固定。
- 运行页内容通过 bottom padding 与末尾控件的 scroll-margin 避让 fixed action surface，首屏不得用大 margin/spacer 推开正常流内容，末尾内容必须可滚动到操作条上方。
- 历史任务、重试配置、阶段选择和帮助文本保持正常文档流，并在手机上以可折叠区域渐进披露。
- 流水线阶段使用单列时间线式列表；运行态、成功、失败和跳过均有文字状态，不依赖颜色。
- 高频运行事件只更新必要节点，不能重建导航或当前聚焦按钮。

### 结果

- 概要与产物文件使用单列列表。
- 节点预览在手机上切换为主次字段卡片：名称/协议/国家为首要信息，地址、速度与可用性为次要信息；桌面保留数据表。
- 长值允许换行和复制，不允许撑破页面。

### 订阅

- 格式选择采用具备 `aria-pressed` 的分段控件，触控高度至少 44px。
- 订阅链接、复制操作和二维码按任务顺序垂直排列。
- 二维码具有 `idle/loading/success/error/unavailable` 状态；失败时提供重试与复制链接替代操作，不能永久显示“生成中”。

### 日志

- 筛选、阶段选择和复制/清空操作分组；危险的清空操作与复制分离，并提供撤销。
- 日志容器使用 `role="log"`。用户接近底部时自动跟随；用户向上阅读时保持位置并显示“有新日志，回到底部”。
- 手机横屏使用动态高度，固定导航与工具栏不能挤占全部内容。

### 设置

- 设置入口继续按主题分组。
- 手机端所有设置使用全屏 sheet；标题区和操作区适配安全区，内容区独立滚动，使用 `100dvh`。
- 数据源和可用性目标在手机上由宽表格切换为字段组列表；每个字段具有“对象名称 + 字段名”的可访问名称。
- sheet 使用真正的模态语义：关闭时不在焦点树中，打开后移动焦点、圈定焦点、支持 Escape，关闭后恢复触发元素。
- 保存操作始终位于软键盘可见区域上方。

## 视觉系统

- 保留现有靛蓝强调色与浅色产品界面，不创建新的品牌主题。
- 移动端以单层 surface 为主，边框承担分组，阴影仅用于固定导航、sheet 与需要浮层关系的元素。
- 正文字号不低于 16px；辅助标签可为 13–14px，但满足 4.5:1 对比度。
- 所有触控目标至少 44×44px，主要操作至少 48px 高。
- 动效仅表达导航选中、sheet 进出与状态变化，时长 150–250ms；遵守 `prefers-reduced-motion`。
- 使用语义化 z-index：sticky、bottom-nav、backdrop、sheet、toast。

## 响应式与设备约束

- `viewport-fit=cover`，全面使用 `safe-area-inset-*`。
- 高度敏感区域使用 `dvh`；为不支持环境保留 `vh` 回退。
- 增加手机横屏规则：`orientation: landscape` 且 `max-height: 500px` 时取消非关键 sticky，压缩日志和页面头部。
- 页面根级别不得横向溢出；只有明确标注的内部滚动区可横向滚动。
- 支持 320、360、375、390、430、720、768px 及手机横屏。

## 可访问性与交互状态

- 导航当前项使用 `aria-current="page"`。
- 分段控件使用 `aria-pressed`；模态 sheet 使用 `role="dialog"`、`aria-modal` 与标题关联。
- 所有按钮具备默认、hover、focus-visible、active、disabled 与 loading 状态。
- 复制操作防重复触发；日志不得记录完整订阅 URL 或节点敏感内容。
- 运行前置校验错误提供持久说明与“去配置”操作，并聚焦对应字段。
- Toast 在手机上清除桌面 `top` 约束，避开安全区和底部导航。

## 测试与验收

### 功能测试

- 390×844：六页导航与每页核心完整流程。
- 360×800：覆盖 380px 断点、导航与顶栏紧凑布局。
- 844×390：手机横屏下运行、日志和设置 sheet。
- 768×1024：平板导航与一至两列布局。
- 设置：四类 sheet 的打开、滚动、编辑、保存、取消、Escape、焦点恢复和软键盘尺寸变化。
- 运行：开始、运行态、停止、重试与高频事件期间焦点稳定。
- 结果、订阅、日志：节点卡片、复制、二维码错误/重试、日志跟随/新消息、清空撤销。

### 布局与可访问性契约

- 根文档 `scrollWidth <= clientWidth`。
- 关键触控目标不小于 44×44px。
- 最后一个可交互元素可滚动到固定导航上方。
- 关闭的 sheet 无可聚焦元素，打开的 sheet 焦点不会进入背景。
- 720/721、380/381、960/961 临界宽度行为明确。

### 视觉回归

- 390×844 下六个页面、运行态、设置 sheet 和登录页均保存 PNG 基线。
- 增加 360px 小屏、768px 平板与手机横屏的关键页面基线。
- H5 先在普通浏览器中完成手动验收，再运行 Electron 验证。
- 移动 E2E 与视觉测试必须进入 PR/发布门禁，不再被 Release CI 排除。

## 完成标准

- 移动端完整功能与桌面一致，没有因屏幕尺寸隐藏核心能力。
- 审查中所有 P1 问题均被修复，P2 问题除有书面合理例外外全部处理。
- 所有目标视口与交互矩阵通过自动化和人工视觉检查。
- 桌面现有单元、E2E 和视觉回归全部通过。

## 运行数据正确性补充

### 去重来源计数

- 每个 canonical 去重节点归属首次观察到它的来源；归属必须持久化到 SQLite，不能只存在于一次运行的内存中。
- 各来源 `deduped_links` 之和必须等于全局 `deduped_links`。
- 正常运行、重试、恢复、旧数据库迁移与 artifact summary 均从同一 SQLite 权威查询生成来源计数。
- 旧 artifact 没有来源去重字段时，界面显示“—/未知”，不得伪造为 0。

### GeoIP 国家识别

- 定位对象是节点配置中的服务器地址，不是通过代理观察到的出口地址。
- 支持 IPv4、IPv6 和域名 A/AAAA 解析；优先使用公开 GeoIP 主供应商，遇到超时、429、非法 schema 或不可用时切换备用供应商。
- 供应商响应必须校验 HTTP 状态和国家代码；成功结果可缓存，失败只允许短 TTL 负缓存。
- 双供应商失败、DNS 失败或未知地址统一返回 `ZZ/其他`，禁止默认或降级为 `US`。
- 正常运行、重试与恢复必须使用同一 GeoIP 服务；UI 继续读取产物中的真实国家标记。
